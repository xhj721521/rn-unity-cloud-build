0. 目标

把“我的”页做成与首页一致的霓虹科技风骨架版：先排好布局与组件，全部占位，后续你再替换背景/插图。

1. 主题与复用

完全复用首页的：卡框样式（霓虹描边+圆角）、按钮、Chip、阴影、动效曲线、数字等宽字体、色板与间距 Tokens。

组件优先复用（以你项目命名为准）：

NeonCard/TileCard（首页功能卡样式）

NeonBorder（描边发光效果）

PrimaryButton / GhostButton

StatusChip（网络、资产）

Carousel（横滑）

ListItem（设置行）

Banner（提示条）

2. 布局（720×1280 竖屏基准）

全局：左右内边距 16；模块上下 20；卡片间距 12；卡片圆角 20；按钮圆角 14；Chip 圆角 999。
入场动效：与首页一致（卡片轻浮、透明 0.9→1）。

顺序（从上到下）：

Hero 身份资产卡（高约 220）

头像(56) + 昵称 + Lv.xx 徽章

右上：网络 StatusChip（● 网络：稳定）

底部：两枚资产 Chip —— ARC、矿石（点击进入资产中心）

右下：邀请队友（主按钮）、导出战报（幽灵按钮）

右上角再放地球图标触发语言切换（如果已有“语言设置”页，直接唤起；否则弹出 ActionSheet）

2×2 指标格（小卡，卡高 84）

矿工（数量） → 点击：矿工管理（沿用现有路由；若无，用 my/miners）

NFT → 点击：仓库-NFT（沿用现有）

地图（已解锁的） → 点击：地图列表（沿用现有）

稀有碎片 → 点击：碎片/合成页（沿用现有）

统一：左上图标(22)，右对齐主值(24, 等宽)，下方副文(12)

地图卡描边同时充当解锁进度环（解锁率=已解锁/总，500ms 补间）

功能入口矩阵（2 列 × 3 行，单卡高 112）

第1行：我的团队｜我的仓库

第2行：我的邀请｜会员中心（角标“非会员/ VIP”）

第3行：战报中心｜收藏亮点

卡样式与首页功能卡一致；暂用低饱和抽象占位图，后面再换图。

已有路由沿用（team / storage / invite / member / reports / highlights 之类）；不存在才新建。

收藏亮点横滑（容器高 120；卡 240×100）

缩略图 + 标题 + 右下 Lv.3 + 右上收藏星

右上“查看全部”链接到现有收藏/仓库相关页

实名提示 Banner（仅未实名显示，56 高）

文案：“未实名，部分功能受限（提现、会员升级等）｜去实名”

设置列表（ListItem）

实名信息（KYC）｜语言 Language｜通知与战报推送｜个性化外观主题｜资金记录

若某项“开发中”，保持样式一致但置灰禁用

底部导航：不改动

3. 交互与路由（已有为先）

资产中心（充值/提现/记录）

入口：Hero 的 ARC/矿石 Chip、设置的“资金记录”。

如果已有钱包/资产页，直达对应 Tab；没有才新增 wallet 页面，含 Topup / Withdraw / History 三个 Tab。

规则：提现需 KYC=已实名；门槛与结算时点从服务端配置（支持“20 USDT 起，18:00 统一结算”）。

会员中心

若已有会员页沿用；无则新增 member。

非会员：灰徽章+“开通会员”；会员：显示 VIP 等级、到期时间、续费/升级。

实名信息（KYC）

若已有 KYC 流程沿用；无则新增 kyc。

状态：未实名/审核中/未通过/已实名；提现按钮按状态禁用/启用。

语言切换

若已有语言设置页沿用；无则弹出 ActionSheet 选择语言（zh-CN/en/vi/ms/th/ar/pt-BR）。

使用 react-i18next 实时切换，不重启应用；写入 AsyncStorage['app.lang']。

4. 指标格数据口径（统一从 Store/接口拿）

miners.total、miners.active（副文：在岗 {active} / {total}，没有则只显示总数）

nft.count（可选 nft.equipped 作副文）

maps.unlocked、maps.total（主值显示 unlocked/total）

shards.rare（≥1000 显示 1.2K 样式）

首次进入拉一次；前台 30s 轮询或事件驱动更新。
